Runtime V2 keeps Taskplane’s durable execution model while replacing the fragile runtime control plane. The architecture explicitly preserves authoritative artifacts like PROMPT.md, STATUS.md, .DONE, and .pi/batch-state.json, so correctness and resumability contracts remain stable. What changes is how live execution is hosted: instead of layered TMUX/session indirection, Runtime V2 moves to deterministic Node process ownership.

The new process tree gives each component a single owner and clear responsibility boundaries. The extension becomes command surface only, the engine owns orchestration policy and lane/merge process lifecycle, lane-runner owns serial per-lane task execution, and agent-host owns direct pi --mode rpc child processes plus telemetry normalization. This removes correctness dependency on terminal infrastructure and replaces sibling file-polling for live state with direct parent-child IPC while still persisting durable history.

Operationally, Runtime V2 introduces normalized runtime artifacts under .pi/runtime/{batchId}/ (agents, lanes, merge, registry) and preserves mailbox-based steering as the canonical control channel. Dashboard visibility is designed around structured event logs and snapshots rather than pane capture, and agent identities remain familiar but are redefined as process-backed IDs rather than TMUX session names. The end state is a more deterministic, recoverable, and observable orchestration runtime, with /orch as the authoritative path.
