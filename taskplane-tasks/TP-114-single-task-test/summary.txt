Runtime V2 keeps Taskplane’s durable file-backed execution model while replacing the fragile runtime control plane. The design intentionally preserves the authoritative artifacts (`PROMPT.md`, `STATUS.md`, `.DONE`, and `.pi/batch-state.json`) and orchestration semantics (DAGs, waves, lanes, worktrees), but removes TMUX and extension-lifecycle coupling from correctness-critical execution.

The architecture introduces deterministic ownership boundaries: extension owns the engine process, engine owns lane-runners and merge hosts, lane-runner owns worker/reviewer agent-hosts, and each agent-host owns exactly one Pi RPC child process. This separation makes failure domains clearer, improves lifecycle management, and replaces file-tail transport hacks with direct parent-child telemetry while still persisting normalized history for dashboards and resume flows.

Operationally, Runtime V2 centers on explicit invariants: resumability, mailbox-only steering, packet-path authority, and no correctness dependency on terminal infrastructure. The end-state is a more stable and observable orchestration runtime where even single-task execution runs through `/orch` semantics, and supervisor/dashboard tooling reads structured runtime artifacts instead of terminal panes.
