Runtime V2 keeps Taskplane’s durable execution model intact while replacing the fragile runtime control plane. The system still treats PROMPT.md, STATUS.md, .DONE, and .pi/batch-state.json as authoritative artifacts for task memory and resumability. What changes is the process topology: TMUX-centric orchestration and sidecar-first telemetry are removed from correctness-critical paths.

The new architecture is explicit about process ownership. The extension is reduced to command surface and supervisor routing, the engine owns orchestration decisions (discovery, waves, resume/abort, merge policy), each lane-runner owns deterministic per-lane execution, and each agent-host owns a direct child pi --mode rpc process. This removes ambiguous lifecycle boundaries and makes failure attribution and cleanup deterministic.

Operationally, Runtime V2 standardizes steering and visibility through mailbox and normalized event streams instead of terminal session coupling. Agent-host handles RPC parsing, telemetry normalization, mailbox injection, and manifest updates, while lane-runner emits lane snapshots for dashboard consumption. The design goal is clear: preserve recoverability and operator clarity while eliminating TMUX dependency from worker/reviewer/merge correctness.
