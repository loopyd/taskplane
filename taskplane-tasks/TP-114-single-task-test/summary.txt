Runtime V2 keeps Taskplane’s durable data plane while replacing its fragile runtime control plane. The architecture intentionally preserves PROMPT.md, STATUS.md, .DONE, and .pi/batch-state.json as authoritative state, but removes operational dependence on TMUX sessions and extension-host lifecycle quirks. The goal is to make execution deterministic and resilient without changing the core user-facing memory model.

The new runtime introduces explicit process ownership boundaries: the extension owns the engine, the engine owns lane-runners and merge hosts, each lane-runner owns worker/reviewer agent-host processes, and each agent-host owns its Pi child process. This simplifies failure attribution, cleanup, and resume behavior because every process has one parent responsible for its lifecycle. It also replaces sibling file-polling telemetry patterns with direct parent-child telemetry flows.

Runtime V2 also standardizes observability and steering around structured artifacts and mailbox messaging. Agent events, manifests, lane snapshots, and registry data are written under a batch runtime directory, and the dashboard reads these normalized files for live status and conversation views. Steering is mailbox-driven (supervisor ↔ agents), packet-home path authority is explicit, and no correctness path depends on terminal infrastructure.
