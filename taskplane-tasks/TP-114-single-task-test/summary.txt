Runtime V2 is a fundamental redesign of Taskplane's runtime control plane. While preserving the existing durable data plane — including PROMPT.md, STATUS.md, .DONE files, batch-state.json, the dependency DAG, waves, lanes, worktrees, and orchestrator branches — Runtime V2 replaces the fragile TMUX-based control infrastructure with a deterministic, process-ownership-based architecture. The core diagnosis motivating this change is that the original runtime suffered from too many overlapping layers (operator session → extension → engine → TMUX lane session → RPC wrapper → Pi task-runner → TMUX worker → RPC wrapper → Pi worker), causing ambiguous ownership, transport inversion, extension-host coupling, and terminal dependency.

The target architecture introduces a clean process hierarchy where every spawned process has exactly one deterministic owner. The extension owns the engine process, the engine owns lane-runners and merge hosts, each lane-runner owns its worker and reviewer agent-hosts, and each agent-host owns its Pi RPC child process. Key new components include the lane-runner process (which replaces the pattern of running lanes as Pi sessions loading task-runner.ts) and the agent-host process (a transport adapter around pi --mode rpc that handles subprocess lifecycle, live RPC event parsing, telemetry normalization, mailbox polling, and conversation persistence). A batch-scoped runtime storage layout under .pi/runtime/{batchId}/ organizes agent manifests, event streams, lane snapshots, and merge state.

Runtime V2 maintains critical invariants: file-backed memory (STATUS.md, .DONE) remains authoritative, batch state stays resumable via batch-state.json, live control uses direct process ownership instead of file polling between siblings, the mailbox is the only supported steering path between operator, supervisor, and agents, and no correctness dependency on terminal infrastructure (TMUX) exists. Success is defined by the complete elimination of TMUX from worker/reviewer/merge lifecycle, dashboard visibility without pane capture, process-registry-backed agent identity, and explicit packet-path authority throughout the execution chain.
