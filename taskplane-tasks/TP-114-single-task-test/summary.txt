Runtime V2 is a redesign of Taskplane's runtime control plane that replaces the
fragile TMUX-based session infrastructure with a deterministic, Node.js-based
process hierarchy. The existing durable data plane — PROMPT.md, STATUS.md,
.DONE files, batch-state.json, the dependency DAG, waves, lanes, worktrees,
and orchestrator branches — all remain intact. What changes is how processes
are spawned, owned, and monitored: instead of deeply nested TMUX sessions
communicating through sidecar files, Runtime V2 introduces a clean ownership
chain where the extension owns the engine, the engine owns lane-runners and
merge hosts, lane-runners own worker and reviewer agent-hosts, and each
agent-host owns its Pi subprocess.

The core architectural principle is that every spawned process has exactly one
deterministic owner. This eliminates the ambiguous ownership, transport
inversion, and terminal dependency problems that plagued the original design.
Key new components include the lane-runner (a deterministic Node process that
manages serial task execution, STATUS parsing, step progression, and checkpoint
discipline for a single lane) and the agent-host (a transport adapter around
Pi RPC that handles subprocess lifecycle, live telemetry normalization, mailbox
polling, and conversation persistence). Steering flows through a structured
mailbox path — the operator talks to the supervisor, the supervisor talks to
agents, and agents reply through outbox channels — with no correctness
dependency on terminal infrastructure.

Runtime V2 also introduces a batch-scoped storage layout under
.pi/runtime/{batchId}/ for agent manifests, event streams, and lane snapshots,
while preserving existing top-level durable files for backward compatibility.
The /task command path is deprecated in favor of /orch as the authoritative
execution path, even for single-task runs. Success is defined by the complete
elimination of TMUX dependencies for worker, reviewer, and merge lifecycle;
dashboard visibility through structured event logs rather than pane capture;
and explicit packet-path authority for all segment and workspace execution.
